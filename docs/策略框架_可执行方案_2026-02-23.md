# 可执行策略框架（V2 文档版，不改代码）

更新时间：2026-02-23
目标：在保留“强者恒强”前提下，降低 UTAD 误入、降低连亏长度、提高策略稳定收益。

## 1. 优化目标与策略定位

建议先走“高胜率稳健型”，再逐步放宽为“高盈亏比型”。

核心目标：
1. 胜率：先从信号质量提升，不靠止损止盈微调。
2. 收益率：通过减少无效交易、保留主升段交易来提升净收益。
3. 稳定性：优先降低最大连续亏损和 UTAD 出场占比。

## 2. 新框架总览（四层 + 打分）

```text
全市场/趋势池
-> L1 方向层（Direction Gate）
-> L2 健康层（Health Gate）
-> L3 维科夫事件层（Event Score）
-> L4 执行层（Execution + Risk）
```

## 3. L1 方向层（只做上行结构）

必选条件：
- `close > MA20 > MA60`
- `MA20` 最近 10 日斜率 > 0
- `ma10_above_ma20_days >= 5`
- `ma5_above_ma10_days >= 3`

作用：确保只在可持续上行结构内找维科夫买点。

## 4. L2 健康层（从“强”改为“稳强”）

### 4.1 过热排除
- `ret_20 > 35%` -> 排除
- `ret_5 / ret_20 > 0.60` -> 排除（末端加速）
- `ATR_percentile_120 > 85%` -> 排除

### 4.2 量价健康
- `up_down_volume_ratio >= 1.2`
- `pullback_volume_ratio <= 1.0`
- 最近 5 日不能出现“放量长上影 + 收绿”密集（>=2 次）

### 4.3 结构稳定
- `price_vs_ma20` 建议在 `[0, +0.06]`
- 最近两次回踩 MA20 都能收回

输出：`health_score (0~100)`。

## 5. L3 维科夫事件评分（替代布尔触发）

定义：

```text
event_score =
  0.30 * background_score
+ 0.25 * position_score
+ 0.25 * volume_price_score
+ 0.20 * confirmation_score
- risk_penalty
```

建议阈值：
- `event_score >= 70` 才允许入场
- `60~70` 进入观察，不下单
- `<60` 直接过滤

### 5.1 背景分（Background）
- 最近 20~60 日是否存在“吸筹->突破->回踩”链条
- 是否处于趋势主升中段，而非末端加速

### 5.2 位置分（Position）
- 事件是否出现在结构关键位（区间边界、回踩支撑）
- 远离关键位的事件降分

### 5.3 量价分（Volume/Price）
- 突破应放量，回踩应缩量
- 若出现量价背离、上影异常，强降分

### 5.4 确认分（Confirmation）
- 事件后 1~3 天必须满足“站稳/转强”
- 不确认不交易

### 5.5 风险惩罚（Risk Penalty）
- `UTAD/SOW/LPSY` 先兆出现时直接扣分
- 扣分上限建议 30 分

## 6. L4 执行层（入场、出场、仓位）

### 6.1 入场
- 默认采用“确认后买入”（T+1 或 T+2）
- 不建议突破当天直接追入

### 6.2 出场
- 结构止损：失守 LPS/Spring/关键支撑位
- 时间止损：`N` 天未转强则离场（例如 8~12 天）
- 风险止盈：出现 UTAD 风险分跃升时分批减仓

### 6.3 仓位与连亏控制
- `max_positions` 保持 8~10
- 连续亏损达到阈值（如 6）时：
  - 降低单笔仓位 20%~40%
  - 临时提高 `event_score` 入场阈值 +5
  - 暂停低等级事件 5 个交易日

## 7. 排序逻辑（关键变更）

现建议排序不再用“涨幅优先”，改为“健康+事件优先”：

```text
final_score = 0.45 * health_score + 0.55 * event_score
```

同日取前 `TopK` 执行。

## 8. 参数起步建议（首轮）

1. `event_score_entry_threshold = 70`
2. `confirmation_delay = 1`（可测试 1/2）
3. `ret_5_over_ret_20_cap = 0.60`
4. `atr_percentile_cap = 85`
5. `time_stop_days = 10`
6. `consecutive_loss_guard = 6`

## 9. 回测验证顺序（必须按顺序）

1. 只加 L2 健康层，不改事件：观察 UTAD 出场占比、连亏。
2. 加事件确认延迟（T+1/T+2）：观察胜率和回撤。
3. 从布尔事件改评分事件：观察收益曲线是否由尖峰变平台。
4. 最后才微调止损止盈。

## 10. 你现有框架上的落地映射（文档级）

- 保留现有四层漏斗骨架。
- 在 Step2/Step3 增加“过热与稳定度”因子（文档规则先行）。
- 将 entry 判定从“事件命中”升级为“事件评分 + 确认”。
- 排序从 `ret40` 主导，改为 `health_score + event_score` 主导。

以上方案不要求立刻改代码，可先按该文档制定实验矩阵，再做小步验证。
