# FinalTrade EXE 功能总说明（V1.2）

适用安装包：`dist/FinalTrade-V1.2.exe`

这份文档不是“只教你怎么启动”，而是完整说明 FinalTrade 这个项目在 EXE 版里到底能做什么、各模块之间如何联动、推荐怎么用。

## 1. 项目整体能力（一句话）

FinalTrade 是一个“从选股到复盘”的一体化工作台，覆盖：

1. 选股漏斗（生成候选池）
2. 待买信号（做入场筛选）
3. 回测验证（待买回测 + 策略回测）
4. 模拟交易（下单/撤单/结算）
5. 持仓管理（看仓位和盈亏）
6. 复盘统计（沉淀交易结论）
7. AI 题材分析（记录与联动）

## 2. EXE 启动方式

1. 双击 `FinalTrade-V1.2.exe`
2. 程序会自动拉起本地后端和前端页面
3. EXE 默认访问地址（前后端同端口）：
   - 页面入口：`http://127.0.0.1:8010`
   - 健康检查：`http://127.0.0.1:8010/health`

如果浏览器没自动打开，手动访问前端地址即可。

补充说明：

- EXE 是“单端口托管”模式，前端静态页面由后端一起提供，所以入口就是 `8010`。
- 如果 `8010` 被占用，程序会自动回退到后续端口（如 `8011`、`8012`），以启动日志显示地址为准。
- `4173` 是开发模式（`start-dev.ps1` / `npm run dev:host`）常用前端端口，不是 EXE 默认端口。

## 3. 顶部菜单（你看到的每个入口在做什么）

### 主流程（选股漏斗）

- 四步漏斗分步运行，逐步收敛股票池。
- 每一步有独立结果池，方便比较与回看。
- 结果可继续流转到待买信号和后续交易链路。

### 策略中心

- 统一维护策略参数与预设模板。
- 选股漏斗 / 待买信号 / 策略回测共用这里的策略配置。
- 目标是避免多页面参数口径不一致。

### 事件判别

- 维护事件评分维度（权重、反向、启用）与触发规则阈值。
- 支持模板“选择 / 应用 / 保存并激活”。
- 当前激活模板会影响信号结果和回测结果。

### 设置

- 配置行情数据路径、数据源、系统参数。
- 配置 AI Provider 等扩展能力。
- 这里是“先检查再开工”的入口。

### 使用说明

- 跳转到在线文档，查看最新操作规范与 FAQ。

## 4. 左侧完整功能地图（项目整体功能）

### 4.1 选股漏斗

- 输入：市场数据 + 当前策略参数。
- 输出：分步骤候选池（用于后续待买信号）。
- 适合做“由宽到窄”的初筛。

### 4.2 待买信号

- 支持趋势池后置模式与全市场扫描。
- 展示阶段评分、事件评分、状态分层（如 Active/Expiring）。
- 可直接跳转 K 线标注页、待买回测、交易页。

### 4.3 待买回测

- 根据当前待买信号快速生成 ETF 回测记录。
- 支持 T+1/T+2 口径、胜率统计、成分股明细。
- 支持持仓 N 天收益率观察（用于评估持仓周期）。
- 支持分页和批量操作。

### 4.4 策略回测

- 对策略在历史区间进行系统回放评估。
- 支持收益平原/参数组合评估、历史回测记录管理。
- 支持导出：
  - `excel/html`：导出当前结果/报告视图。
  - `ftbt`：导出完整回测包，便于导入与共享。

### 4.5 模拟交易

- 按 A 股 T+1 规则模拟交易。
- 支持下单、撤单、补结算、重置。
- 支持待买单集中管理与批量提交。
- 仓位方式支持 `lots` / `amount` / `position(%)`。

### 4.6 持仓管理

- 查看持仓结构、资金分布、浮盈浮亏。
- 支持快捷卖出，形成交易闭环。

### 4.7 复盘统计

- 支持交易结果统计（买入/卖出日期轴）。
- 支持日复盘、周复盘、标签统计。
- 支持 Excel / CSV / PDF 导出。

### 4.8 复盘分享

- 搜索股票并生成复盘分享卡片。
- 适合做复盘对外交流或团队同步。

### 4.9 资讯面板

- 聚合复盘期市场资讯。
- 支持 24/48/72 小时时效窗口刷新。

### 4.10 AI 分析

- 记录 AI 题材分析历史。
- 支持删除、筛选，并与个股 K 线标注页联动。

### 4.11 个股图表（K线标注页）

- 展示 K 线、分时和标注信息。
- 用于信号核验、人工复盘与注释沉淀。

## 5. 推荐使用方式（实战流程）

### 流程A：日常选股与执行

1. 设置里确认数据源正常
2. 主流程跑漏斗
3. 待买信号筛选候选
4. 待买回测检查短周期表现
5. 模拟交易提交执行
6. 持仓管理跟踪

### 流程B：策略迭代

1. 在策略中心调整参数
2. 在事件判别调整维度/规则
3. 跑策略回测验证改动效果
4. 导出报告沉淀版本

### 流程C：收盘复盘

1. 在复盘统计记录日/周结论
2. 用复盘分享输出卡片
3. 用 AI 分析补充题材记录

## 6. 导出、共享、导入说明

- `excel/html`：当前页面结果导出，适合阅读和汇报。
- `ftbt`：完整回测包，适合跨机器共享和报告导入。
- 建议命名规范：`策略名_日期_版本号`。

## 7. 关键概念（避免误解）

### 事件判别里的“反向/启用”

- `反向`：该维度按 `100 - 原始分` 参与总分。
- `启用`：关闭后该维度不计入总分。

### “排队中”不一定代表有其他任务

- 可能是状态同步延迟或任务状态未及时回收。
- 先刷新列表；必要时重启程序后重试。

### 为什么会只看到测试数据

- 多数是误开了 Mock 模式（例如 `VITE_ENABLE_MSW=true`）。
- 关闭 Mock 并重启后，才会看到真实后端数据。

## 8. 本地文件与数据位置

- 应用状态：`~/.tdx-trend/app_state.json`
- 模拟交易状态：`~/.tdx-trend/sim_state.json`
- 运行日志目录：`runtime-logs/`

## 9. 常见故障排查

### 页面打不开

- 先检查：`http://127.0.0.1:8010/health` 是否返回正常。
- 再确认前端地址：`http://127.0.0.1:8010`。

### 接口 404 / 加载失败

- 检查后端是否正常运行。
- 检查设置页中的数据路径与数据源。

### 数据结果和预期不一致

- 先统一策略参数、事件模板、回测区间。
- 再做横向对比，避免混口径结论。

### 程序状态异常

- 完全退出 EXE 后重新启动。
- 如仍异常，结合 `runtime-logs/` 定位问题。
