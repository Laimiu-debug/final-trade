# Final Trade 使用说明（V1.2）

本说明用于帮助新用户从零开始完成 Final Trade 的选股、信号、回测、交易与复盘闭环。

## 快速开始

Windows 一键启动：

```powershell
.\start.bat
```

或：

```powershell
.\start-dev.ps1
```

默认地址：

- 前端：`http://127.0.0.1:4173`
- 后端健康检查：`http://127.0.0.1:8010/health`

首次使用建议：

1. 先跑一轮选股，确认有结果。
2. 再进入待买信号、回测、交易、复盘逐页验证流程。

## 页面导航

顶部菜单说明：

- `主流程`：进入选股漏斗与主数据入口。
- `策略中心`：统一管理策略参数、策略模板和能力开关。
- `事件判别`：配置事件评分维度与触发规则（影响信号与回测）。
- `设置`：配置数据源、路径、系统行为和环境开关。
- `使用说明`：查看当前操作文档与 FAQ。

## 标准操作流程（从选股到复盘）

1. 在“主流程”完成选股，形成候选池。
2. 进入“待买信号”筛选并确认候选标的。
3. 在“待买回测/策略回测”验证策略表现（收益、回撤、胜率、稳定性）。
4. 将拟执行标的加入“模拟交易”下单并跟踪成交。
5. 在“持仓管理”观察仓位、盈亏和资金分布。
6. 在“复盘统计”完成日复盘/周复盘并沉淀结论。
7. 需要时在“AI 分析”补充题材与个股分析记录。

## 重点功能说明

### 策略中心

- 建议把参数管理集中在策略中心，避免多页面口径不一致。
- 调整参数后，建议重新跑信号与回测验证一致性。

### 事件判别

- 维度总分为加权计算结果。
- `反向`：该维度按 `100 - 原始分` 参与计算。
- `启用`：关闭后该维度不参与最终总分。

### 待买回测

- 支持按持仓天数口径观察收益。
- 建议固定参数后进行横向比较，避免混合口径。

### 策略回测

- 用于评估整体策略表现与稳定性。
- 支持导出结果用于复盘、归档和共享。

### AI 分析

- 用于补充记录与辅助理解。
- 不建议替代规则验证与回测结论。

## 导出与共享

- `excel/html`：导出当前结果或报告，便于阅读、归档、汇报。
- `ftbt`：导出完整回测包（可用于导入/共享）。

命名建议：

- `策略名 + 日期 + 版本号`，便于追溯和回滚。

## 常见问题（FAQ）

### 1) 显示“排队中”，但没有其他任务在运行

- 先刷新任务列表并等待几秒。
- 检查是否有历史任务状态未回收。
- 必要时重启后端服务再重试。

### 2) 页面里只看到测试数据

- 检查是否启用了 Mock 模式（如 `VITE_ENABLE_MSW=true`）。
- 关闭 Mock 后重启前端，确认请求已连接真实后端。

### 3) 事件判别里的“反向/启用”是什么意思

- `反向`：翻转该维度分数方向后参与加权。
- `启用`：控制该维度是否参与总分。

## 数据与文件位置

- 应用状态：`~/.tdx-trend/app_state.json`
- 模拟交易状态：`~/.tdx-trend/sim_state.json`
- 运行日志目录：`runtime-logs/`

## 故障排查

- 页面打不开：先检查后端健康接口是否可访问。
- 请求 404：确认前端代理目标与后端端口一致。
- 数据异常：检查数据源配置并执行数据同步。
- 结果不一致：统一策略参数、事件模板、日期范围后再对比。
