# 当前策略框架梳理（基于代码）

更新时间：2026-02-23
范围：仅基于当前仓库代码，不含主观优化假设。

## 1. 回测执行路径（当前真实行为）

当前回测存在两条路径：

1. 矩阵引擎路径（默认优先）
- 开关来源：`backend/app/store.py:5207`
- 默认值：开启（`backend/app/store.py:503`, `frontend/src/pages/settings/SettingsPage.tsx:212`）
- 若开启且模式支持（`trend_pool/full_market + daily/weekly/position`），优先走矩阵回测（`backend/app/store.py:7765`）

2. 旧路径（逐股逐日 snapshot）
- 当矩阵引擎关闭或失败时回退（`backend/app/store.py:7794`）
- 旧路径会按事件日期逐日匹配 entry/exit 事件（`backend/app/core/backtest_engine.py:473`）

结论：默认情况下，你的回测不是逐日完整维科夫事件执行，而是矩阵简化信号执行。

## 2. 趋势选股四层筛选（trend_pool）

核心函数：`backend/app/store.py:4144`

### Step1 流动性/活跃度 + 强势截断
触发条件：
- `turnover20 >= turnover_threshold`
- `amount20 >= amount_threshold`
- `amplitude20 >= amplitude_threshold`
- 按 `ret40` 降序排序后取 `top_n`
- 额外保护上限：最多 400（`backend/app/store.py:4161`）

### Step2 趋势结构与回撤约束
触发条件：
- `retrace20` 在区间 `[0.05, 0.25]`（loose 模式有小幅放宽）
- `pullback_days <= 3`（loose 可 +1）
- `ma10_above_ma20_days >= 5`（loose 可 -1）
- `ma5_above_ma10_days >= 3`（loose 可 -1）
- `abs(price_vs_ma20) <= 0.08`
- `price_vs_ma20 >= 0`
- `trend_class != 'B'`

### Step3 量价健康与风险剔除
触发条件：
- `vol_slope20 >= 0.05`
- `up_down_volume_ratio >= 1.3`
- `pullback_volume_ratio <= 0.9`
- 非 `has_blowoff_top`
- 非 `has_divergence_5d`
- 非 `has_upper_shadow_risk`
- 非 `degraded`

### Step4 AI 置信度与主题阶段
触发条件：
- `ai_confidence >= 0.55`
- `theme_stage in ('发酵中', '高潮')`
- 按 `score + ai_confidence*20` 排序，取前 8

### Step 输出选择
由 `trend_step` 决定最终使用 step1/2/3/4 哪一层（`backend/app/store.py:4125`）。
`auto` 默认取 `step4`，若为空回退 `step3`。

## 3. 滚动股票池

核心函数：`backend/app/store.py:4903`

- 支持 `daily/weekly/position` 三种滚动
- 每个刷新日按上述四层筛选生成当日可交易池
- 全区间按 `payload.max_symbols` 限制当日标的上限
- 回测执行时按日期校验 symbol 是否在当日允许池中

## 4. 维科夫事件触发标准（完整版，旧路径/信号计算使用）

核心函数：`backend/app/core/signal_analyzer.py:200`

说明：以下均为代码中的“触发条件摘要”，不是教科书定义。

### 4.1 吸筹侧事件

1. `PS`
- 最近 `SC` 之前 25 日内
- `tr_pos<=0.42` 且 `vol_ratio>=1.08` 且 `close<=MA*1.02`

2. `SC`
- 近 35 日优先找低位放量衰竭 K
- 关键条件：收盘靠近低位 + `tr_pos<=0.55` + `vol_ratio>=1.05`

3. `AR`
- `SC` 后最多 18 日内
- 从 `SC` 收盘反弹幅度 >= 5%

4. `ST`
- `SC/AR` 后窗口内
- 低点接近 `SC` 低点（误差<=4%）且量能低于 `SC` 的 85%

5. `TSO`
- `ST/AR/SC` 后
- 跌破前低后收回 + `vol_ratio>=1.0`

6. `Spring`
- 形态窗口内
- 跌破前低 1.5% 后收回 + `vol_ratio>=1.15`

7. `SOS`
- `Spring/TSO/ST/AR/SC` 后
- `close > MA*1.01` + `10日收益>5%` + `vol_ratio>=1.05`

8. `JOC`
- `SOS` 后或同窗口
- 收盘>=前 20 日高点*1.005 + `vol_ratio>=1.2`

9. `LPS`
- 以 `SOS/JOC` 为锚点后的 12 日内
- `close > MA*0.995` + 缩量（低于前 5 日均量的 95%）且回踩不超过 6 天

### 4.2 派发/风险侧事件

1. `PSY`
- `BC` 前 20 日内
- 高位 (`tr_pos>=0.62`) + 放量 + `close>=MA*0.98`

2. `BC`
- 近 35 日高位放量候选
- 收盘靠近高位 + `tr_pos>=0.58` + `vol_ratio>=1.05` + 接近阶段新高

3. `AR(d)`
- `BC` 后 18 日内
- 从 `BC` 收盘回落 >= 6%

4. `ST(d)`
- `BC/AR(d)` 后窗口内
- 反抽接近 `BC` 高点（误差<=4%）+ 缩量 + 收盘偏弱

5. `UTAD`
- `ST(d)` 或 `BC` 后
- 高点创新高（前高*1.01）但收盘跌回前高下方 + 上影>=50% + `vol_ratio>=1.2`

6. `SOW`
- `UTAD` 或 `AR(d)` 后
- `10日收益<=-5%` + `close < MA*0.995` + `vol_ratio>=1.1`

7. `LPSY`
- `SOW/UTAD` 后 16 日内
- `close < MA` 且最近 5 日高点低于更早窗口高点（低高点结构）

### 4.3 评分与相位

- `entry_quality_score` 由 phase/event/structure/trend/volatility 加权得到（`backend/app/core/signal_analyzer.py:668`）
- `risk_events` 会扣分（例如 `UTAD -10`）
- `signal_priority` 优先级：`LPS > JOC > SOS > Spring ... > UTAD ...`（`backend/app/core/signal_analyzer.py:691`）

## 5. 回测入场/出场标准（旧路径）

核心函数：`backend/app/core/backtest_engine.py:434`

### 入场
- 当日 `event_dates[event] == as_of_date` 且事件在 `payload.entry_events`
- 过滤：
  - `event_count >= min_event_count`
  - `require_sequence=true` 时必须 `sequence_ok`
  - `entry_quality_score >= min_score`
- 成交：下一交易日开盘价买入

### 出场
- 先后顺序：止损 -> 止盈 -> 事件出场 -> 最大持有天数 -> 区间末平仓
- 事件出场来源：`payload.exit_events` 在当天触发

## 6. 矩阵信号标准（默认回测路径）

核心函数：`backend/app/core/backtest_signal_matrix.py`

### 池内条件 `s1~s4`
- `s1`: 20/60 振幅比（ATR 代理）< 0.7
- `s2`: 10/60 均量比 < 0.6
- `s3`: 40 日涨幅 TopN（行内排名）
- `s4`: 横盘收敛（20 日振幅<0.18 且贴近 MA20）
- `in_pool`: `s1+s2+s3+s4 >= 2`

### 买卖信号
- `buy_signal = (s5 or s6) and in_pool`
- `s5`（类似 SOS）：突破前 20 日高点 + 放量
- `s6`（类似 LPS）：上穿 MA10 + 站稳低点
- `sell_signal = s8 or s9`
- `s8`: 跌破前 20 日低点或跌破 MA20*0.97
- `s9`: 跌破 MA10 且放量且 40 日收益为负

重要差异：矩阵路径里并未逐条使用完整维科夫事件链（PS/SC/AR/...）；主要是 `s5/s6/s8/s9` 的简化映射。

## 7. 收益平原（plateau）当前行为

核心函数：`backend/app/store.py:6679`

- 支持并行评估，线程数来自：
  - 配置 `backtest_plateau_workers`（默认 4）
  - 环境变量 `TDX_TREND_BACKTEST_PLATEAU_WORKERS` 可覆盖
- 每个参数点评估时，强制 `enable_advanced_analysis=false`（`backend/app/store.py:6854`）
- 所以：回测若勾选高级分析，不代表收益平原每个点评估也做高级分析；平原默认关闭高级分析以提速。

## 9. 现状结论（不含优化方案）

1. 你的“趋势四层漏斗”在代码中是存在且生效的。
2. 完整维科夫事件判定实现是齐全的（16 个事件）。
3. 但默认回测优先走矩阵简化信号路径，事件执行逻辑与完整版并不完全等价。
4. 收益平原默认并行，且默认关闭高级分析。
