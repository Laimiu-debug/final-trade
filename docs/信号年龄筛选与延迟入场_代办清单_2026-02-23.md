# 信号年龄筛选与延迟入场（T+2/T+3）代办清单

更新时间：2026-02-23  
状态：待实现（仅文档，不改代码）

## 1. 目标

在“信号筛选”和“回测执行”两端同时支持以下能力：

1. 按“信号已出现天数”筛选信号（例如只看已出现 2~3 个交易日的信号）。  
2. 回测支持延迟入场（T+2 / T+3，以交易日为准）。  
3. 延迟期间若出现风险破坏条件，则取消该笔入场。

## 2. 范围

1. 信号接口与筛选参数。  
2. 回测参数与执行引擎（矩阵路径 + 旧路径保持一致）。  
3. 前端设置与回测页面参数输入。  
4. 测试与回测验证。

## 3. 代办清单

## 3.1 参数与契约

- [ ] 在信号查询参数中新增：`signal_age_min`、`signal_age_max`（单位：交易日）。  
- [ ] 在回测请求参数中新增：`entry_delay_days`（取值建议：1~5，默认 1）。  
- [ ] 统一接口文档与类型定义（后端 `models` / 前端 `contracts`）。

## 3.2 信号筛选（Signals）

- [ ] 计算 `signal_age_days = as_of_date - trigger_date`（交易日差，不用自然日）。  
- [ ] 支持按 `signal_age_min <= signal_age_days <= signal_age_max` 过滤。  
- [ ] 返回字段补充 `signal_age_days`，便于前端展示与排序。  
- [ ] 当 `as_of_date` 缺失时，定义默认行为并写入说明（建议按最新交易日计算）。

## 3.3 回测入场（Backtest）

- [ ] 将入场索引从固定 `signal_index + 1` 改为 `signal_index + entry_delay_days`。  
- [ ] 旧路径与矩阵路径都使用同一套 `entry_delay_days` 语义。  
- [ ] 若延迟后超出样本区间，按“无可入场日”处理并计入跳过原因。  
- [ ] 保持现有 `enforce_t1` 语义不冲突（`entry_delay_days>=1` 时天然满足）。

## 3.4 延迟期间失效保护（关键）

- [ ] 定义“延迟失效窗口”：`signal_date+1` 到 `entry_date-1`。  
- [ ] 若窗口内出现 `UTAD/SOW/LPSY` 或卖出信号，取消该笔入场。  
- [ ] 将取消原因写入回测 notes / skip reason（如 `delay_invalidated_by_risk_event`）。

## 3.5 前端交互

- [ ] 在“信号页”增加“信号已出现天数”筛选输入。  
- [ ] 在“回测页”增加“延迟入场天数”参数（默认 1，可选 2/3）。  
- [ ] 在参数摘要与导出报告中展示 `entry_delay_days` 与信号年龄过滤条件。  
- [ ] 在帮助文案中明确“按交易日计算”。

## 3.6 测试与验收

- [ ] 单元测试：信号年龄计算正确（含停牌/缺失交易日场景）。  
- [ ] 单元测试：`entry_delay_days=1/2/3` 的入场日与价格正确。  
- [ ] 单元测试：延迟窗口出现风险事件会取消入场。  
- [ ] 集成测试：矩阵路径与旧路径在同参数下行为一致。  
- [ ] 回测对照测试（A/B）：
  - A：`entry_delay_days=1`
  - B：`entry_delay_days=2`
  - C：`entry_delay_days=3`
  - 统计：胜率、总收益、最大回撤、最大连亏、UTAD 出场占比、交易数。

## 4. 验收标准（Done Definition）

1. 能在信号页筛选“已出现 N 天”的信号，并正确展示。  
2. 回测可选择 T+2/T+3 入场，且交易明细中的 `signal_date/entry_date` 对应正确。  
3. 延迟期间风险破坏会阻断入场，结果中可追踪原因。  
4. 回测导出与任务记录包含新增参数。  
5. 关键测试通过，且矩阵/旧路径口径一致。

## 5. 备注

1. 建议先上 `entry_delay_days=2` 作为默认实验组，再扩展到 `3`。  
2. 若目标是提胜率，优先观察：`UTAD 占比` 与 `最大连续亏损` 的改善幅度。  

## 6. 趋势选股升级事项（稳强优先）

### 6.1 漏斗维度升级
- [ ] 在方向层之外新增“趋势健康层”（Health Gate），避免仅按涨幅速度筛选。
- [ ] 明确“强势”与“过热”是两套指标：强势用于入池，过热用于降权/拦截。

### 6.2 强势定义（保留强者恒强）
- [ ] 采用中周期强势定义（如 ret40/ret60 + 均线结构）作为强势主轴。
- [ ] 保留 `close > MA20 > MA60` 与 MA20 上行的方向约束。
- [ ] 增加“最近 N 日在 MA20 上方天数”作为结构稳定项。

### 6.3 过热与末端加速处理
- [ ] 增加末端加速识别：`ret5/ret20`（如 >0.6）作为过热信号。
- [ ] 增加波动失控识别：ATR/Price 分位阈值（如 >85%）。
- [ ] 规则采用“先降权、后剔除”：仅多项同时触发时硬过滤，避免误伤龙头。

### 6.4 趋势稳定度评分
- [ ] 新增 slope_stability（涨幅分布均匀度/单日涨幅标准差）。
- [ ] 新增 volatility_stability（ATR 分位、振幅收敛度）。
- [ ] 新增 pullback_quality（最近两次回踩 MA20 的站回质量）。
- [ ] 输出 `health_score (0~100)` 并用于同日排序。

### 6.5 排序与执行
- [ ] 将“涨幅优先排序”调整为“健康分 + 事件分”综合排序。
- [ ] 评估同日 TopK 规则，避免高风险票在同日集中占用名额。

### 6.6 趋势选股验收
- [ ] 对比 Top1-50 / 51-100 / 101-200 三组胜率与 UTAD 占比。
- [ ] 对比“仅涨幅排序”vs“健康+事件排序”的回测结果。
- [ ] 验证新增健康层后最大连亏与回撤是否下降。

## 7. 维科夫事件升级事项（结构+确认）

### 7.1 事件判定框架升级
- [ ] 将事件触发从“布尔命中”升级为“背景+位置+量价+确认”四维判定。
- [ ] 为每个事件输出分项得分与总分，支持分级（A/B/C）。
- [ ] 仅允许 A 级事件触发入场，B 级观察，C 级过滤。

### 7.2 关键事件确认机制
- [ ] SOS：突破后 1~3 日站稳（收盘站上突破位）才允许入场。
- [ ] LPS：回踩缩量 + 转强 K 线确认后才允许入场。
- [ ] Spring：次日不破 Spring 低点并出现转强确认。
- [ ] JOC：突破后不快速跌回前高下方（反抽确认）。

### 7.3 风险事件前置拦截
- [ ] 将 `UTAD/SOW/LPSY` 从“事后出场”前移为“事前禁入/降权”。
- [ ] 设置风险评分阈值：超过阈值时禁止新开仓或降低仓位。
- [ ] 延迟入场窗口内出现风险事件时，强制取消该笔交易。

### 7.4 入场时机实验
- [ ] 增加 A/B/C 入场实验：事件当天、T+1 确认、T+2/T+3 确认。
- [ ] 按事件类型分别统计确认延迟对胜率与收益的影响。

### 7.5 维科夫升级验收
- [ ] 统计 `exit_reason=UTAD` 占比是否显著下降（目标：下降 30%+）。
- [ ] 统计最大连续亏损是否下降（目标：明显缩短）。
- [ ] 在收益不显著恶化前提下，胜率与稳定性指标提升。

## 8. 一致性与口径治理（必须项）

## 11. 结构性盲区修复事项（P0/P1/P2）

目标：补齐当前 v2 升级中的关键盲区，优先解决会导致“高频假信号”的架构问题。

### 11.1 P0（优先级最高）

- [ ] 事件序列合法性（soft gate）  
说明：事件判定不再是孤立单点，需检查前置事件覆盖率（如 SOS 前应有 ST/Spring 背景）。  
验收：孤立 SOS/JOC 的误触发率显著下降，UTAD 回撤型交易减少。

- [ ] 时间衰减（event decay）  
说明：事件分数按 `days_since_event` 衰减，避免“僵尸信号”持续加分。  
验收：旧事件驱动的迟到入场占比下降，信号时效性提升。

- [ ] 路径语义对齐（matrix/legacy）  
说明：矩阵路径与旧路径在事件语义、确认逻辑、风险拦截上保持一致。  
验收：同参数下两路径结果差异可控，核心统计口径一致。

### 11.2 P1（精度打磨）

- [ ] K 线微观确认（candle_quality）  
说明：对 SC/Spring/UTAD 等事件引入实体比例、影线比例、收盘位置子分数。  
验收：单事件判定精度提升，噪声信号减少。

- [ ] 量的日历效应修正（calendar-adjusted volume baseline）  
说明：量比基准从简单 20 日均量升级为“同星期几基准”或 z-score，并处理长假窗口。  
验收：节前节后异常量触发率下降。

### 11.3 P2（有数据再上）

- [ ] 筹码迁移代理（distribution vs shakeout）  
说明：引入简化成本重心迁移，区分“高位派发”与“高位洗盘”。  
验收：BC/UTAD/SOW/LPSY 风险识别提前，误杀趋势股减少。

- [ ] 多周期交叉验证（weekly_context）  
说明：周线趋势/位置作为日线事件乘数，提高高置信度信号占比。  
验收：高分信号胜率提升，低质量日线逆趋势信号减少。

### 11.4 里程碑建议（盲区修复）

- [ ] M8：完成 P0 三项（序列合法性、时间衰减、路径语义对齐）。
- [ ] M9：完成 P1 两项（K线微观确认、量的日历效应修正）。
- [ ] M10：在数据条件满足时完成 P2 两项（筹码迁移、多周期交叉）。
