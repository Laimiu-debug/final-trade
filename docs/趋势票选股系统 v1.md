# 趋势票选股系统 v1.1

<aside>
🎯

**定位**：趋势中线选股系统 v1.1，从**已经启动的强势票**中筛选，用图形判断位置、量能判断持续性、题材判断空间。

**核心思路**：让市场先用资金投票（涨幅排名），再从结果中精选——不预测启动，只跟随趋势。

**与 v1.0 的区别**：v1.0 涨幅范围太窄（20日 20-100%）且无活跃度过滤；v1.1 用 1-2 月涨幅前 500 做入口，大幅缩小范围的同时保证流动性。

</aside>

---

## 一、v1.0 → v1.1 核心变化

| **维度** | **v1.0（旧）** | **v1.1（新）** |
| --- | --- | --- |
| **入口逻辑** | 20 日涨幅 20–100%，取前 300 | 1–2 月涨幅排名前 500，让市场先筛一遍 |
| **活跃度** | 无要求，会选到僵尸票 | 日均换手 ≥ 5%，日均成交额 ≥ 5 亿 |
| **趋势定位** | 无，经常选到主升浪末期 | 看图形判断启动日期，按涨幅分档定位早/中/晚期 |
| **持续性判断** | 仅看上涨日量 > 下跌日量 | 量能趋势 + 题材发酵阶段双重验证 |
| **板块过滤** | 第 5 层排序参考 | 题材逻辑作为核心筛选维度 |
| **持仓规则** | 固定止盈止损 | 区分 A 类（MA10 跟踪）和 A 偏 B（MA5 跟踪加速段） |

---

## 二、筛选漏斗（4 步）

### 第 1 步：涨幅入口——缩小范围

<aside>
📊

**逻辑**：近 1–2 月涨幅排名前 500，等于让全市场资金帮你做第一轮筛选。能涨上来的票，流动性、资金关注度、板块热度天然有保障。

</aside>

```python
输入：全市场 A 股（非 ST、上市 > 250 天）
步骤：
  1. 计算每只票近 40 个交易日（约 2 个月）的累计涨幅
  2. 按涨幅降序排列
  3. 取前 500 只

活跃度硬门槛（在前 500 中再过滤）：
  - 近 20 日日均换手率 ≥ 5%
  - 近 20 日日均成交额 ≥ 5 亿元
  - 近 20 日日均振幅 ≥ 3%

输出：约 300–400 只 → 活跃强势池
```

### 第 2 步：看图形——判断启动日期与趋势位置

<aside>
📈

**逻辑**：从 K 线图上识别趋势的启动点在哪里，当前处于什么阶段。**这一步需要人工看图**，量化指标只做辅助快筛。

</aside>

**量化辅助快筛**（帮你从 300+ 只快速缩到 50–80 只）：

```python
输入：第 1 步结果
快筛条件：
  - 均线多头排列：MA5 > MA10 > MA20，收盘价 > MA20
  - 趋势未走完：MA5 未下穿 MA10（排除已见顶回落的票）
  - 回调中或刚启动优先：当前价距近 20 日最高点回撤 5–25%
    （正在回踩的票比刚创新高的票买点更好）

输出：约 50–80 只 → 待看图池
```

**人工看图判断**（核心环节）：

逐票看 K 线，回答三个问题：

1. **启动日期在哪？**
    - 找到从横盘/下跌转为放量上涨的那根 K 线
    - 标记启动价和启动日期
2. **当前处于什么阶段？**

| **阶段** | **从启动日涨幅** | **K 线特征** | **操作建议** |
| --- | --- | --- | --- |
| 🟢 早期 | < 30% | 刚突破横盘，均线刚排好 | **重点关注，等回踩买入** |
| 🟡 中期 | 30–80% | 阶梯上涨中，均线多头发散 | **可以做，等明确回踩信号** |
| 🔴 晚期 | > 80% | 加速拉升或高位震荡 | **不做，观察为主** |
1. **走势类型是什么？**
    - **A 类（阶梯慢牛）**：小阳线为主，回调浅，贴均线 → ✅ 重点做
    - **A 偏 B（慢牛带加速）**：前半段 A 类，后半段开始放量加速 → ✅ 可以做
    - **B 类（脉冲涨停牛）**：涨停驱动，回调深，量能脉冲 → ❌ 不做

```
输出：约 15–30 只 → 形态确认池
（标注：启动日期、当前阶段、走势类型）
```

### 第 3 步：看量能——判断持续性

<aside>
📊

**逻辑**：趋势能不能继续，量能说了算。量能持续放大 = 资金持续流入 = 趋势大概率延续。

</aside>

对第 2 步留下的 15–30 只票，检查以下量能信号：

```python
✅ 量能健康（保留）：
  - 近 20 日量能整体递增（vol_slope > 0）
  - 上涨日量能 > 下跌日量能 × 1.3（资金做多意愿强）
  - 回调时明显缩量（回调日量 < 5 日均量 × 0.7）
  - 没有出现「天量见天价」（最近一次最大量柱后，价格仍在创新高）

❌ 量能衰竭（排除）：
  - 最近 5 日量能持续萎缩但价格还在涨（量价背离）
  - 出现过巨量长上影线（主力出货信号）
  - 高位放巨量但收阴线或十字星（多空分歧严重）

输出：约 8–15 只 → 量能确认池
```

### 第 4 步：看题材——判断上涨空间

<aside>
🔥

**逻辑**：A 股趋势票几乎都有板块/题材逻辑支撑。题材还在发酵中 = 后续还有资金愿意进来 = 趋势有空间延续。

</aside>

对第 3 步留下的 8–15 只票，判断题材状态：

```python
✅ 题材仍在发酵（保留）：
  - 所属板块近 5 日仍有涨停股出现（板块热度未退）
  - 有持续的政策/事件催化（如 AI、机器人、资源涨价等）
  - 同板块有新的补涨票启动（说明资金还在板块内流转）
  - 板块内龙头尚未见顶（龙头不倒，跟风票就有空间）

❌ 题材已过高潮（排除）：
  - 板块近 5 日无涨停，涨幅开始回落
  - 龙头票已经高位放量滞涨或开始回调
  - 市场主线已切换到其他板块
  - 利好已完全兑现，无新的催化预期

优先级排序：
  1. 当期市场主线板块中的 A 类票（最优）
  2. 次主线板块中的 A 类票
  3. 主线板块中的 A 偏 B 票

输出：5–8 只 → 最终待买清单
```

## 三、买点规则

### 买点 A：回踩均线买入（主要买点）

```python
触发条件：
  - 收盘价回踩至 MA10 附近（MA10 × 0.98 ~ MA10 × 1.02）
  - 回调缩量（当日量 < 5 日均量 × 0.7）
  - 收盘价仍 > MA20（未破 20 日线）
  - 回调天数 ≤ 3 天
动作：次日集合竞价或开盘买入
```

### 买点 B：突破新高回踩确认

```python
触发条件：
  - 近 3 日内放量突破前高（量 ≥ 5 日均量 × 1.5）
  - 突破后回踩不破突破位（回踩幅度 < 3%）
  - 回踩日缩量
动作：回踩确认当日尾盘或次日买入
```

### 买点 C：板块分歧转一致

```python
触发条件：
  - 板块经历 1–2 天分歧调整（板块跌幅 > 2%）
  - 目标票在分歧中抗跌（跌幅 < 板块平均跌幅的 50%）
  - 板块出现转一致信号（次日板块涨停家数回升）
动作：转一致确认后买入
```

## 四、持仓与卖出规则

### A 类持仓模式（默认）

<aside>
📋

**核心原则**：用 MA10 做持仓锚，少动拿趋势。

</aside>

```python
止损：
  - 收盘价 < MA20 × 0.97 → 次日开盘卖出
  - 或：亏损 ≥ 8% → 次日开盘卖出

持仓跟踪（MA10 标准）：
  - 只要收盘价 > MA10 → 继续持有
  - 连续 2 日收盘价 < MA10 → 减仓 1/2
  - MA5 下穿 MA20（均线死叉）→ 清仓

止盈（分批）：
  - 盈利 20% → 卖出 1/3
  - 盈利 40% → 再卖 1/3
  - 剩余仓位 MA10 跟踪，破则清
```

### A 偏 B 加速模式（切换条件）

<aside>
🚀

**切换信号**：持仓期间出现加速特征时，从 MA10 跟踪切换到 MA5 跟踪，给加速段更多空间。

</aside>

```python
切换条件（满足任意 2 条）：
  - 单日涨幅 > 5%（之前中位数 1–3%）
  - 当日量能 > MA10 量能 × 2
  - MA5 与 MA10 间距突然扩大至 > 3%
  - 板块当天 ≥ 3 只票涨幅 > 5%（板块共振）

切换后规则：
  - 持仓跟踪从 MA10 改为 MA5
  - 连续 2 日收盘价 < MA5 → 减仓 1/2
  - MA5 下穿 MA10 → 清仓
  - 不主动止盈，让利润奔跑
```

## 五、每周执行流程

| **时间** | **动作** | **说明** |
| --- | --- | --- |
| **周末** | 跑涨幅排名 | 导出近 40 日涨幅前 500 + 活跃度过滤，得到 300–400 只 |
| **周末** | 快筛 + 看图 | 量化快筛到 50–80 只，逐票看图标注启动日、阶段、类型 |
| **周末** | 量能 + 题材 | 检查量能健康度和题材发酵阶段，精选 5–8 只进待买清单 |
| **盘中** | 等买点 | 只在待买清单的票出现买点 A/B/C 时操作，其他时间不动 |
| **每日盘后** | 持仓检查 | 检查持仓票的 MA10/MA5 状态，是否触发减仓/清仓/模式切换 |
